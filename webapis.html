URI/URL: Identificación de Recursos

Definición

Un URI (Uniform Resource Identifier) es una cadena que identifica un recurso en la web de manera única.

Componentes

Parte Ejemplo
esquema https, ftp, mailto
host api.ejemplo.com
ruta /v1/usuarios
query ?page=2&sort=name
Ejemplo Completo
https://admin:pass123@api.ejemplo.com:8080/v1/users?role=admin#section2
Casos Especiales
  • Puertos no estándar: :8080, :3000
  • Codificación URL: Espacios → %20
  • DocumentRoot: /var/www/html

API REST: Estilo Arquitectónico

Principios Fundamentales

  • Identificación única de recursos
  • Manipulación a través de representaciones
  • Mensajes autodescriptivos
  • Hipermedia como motor de estado (HATEOAS)

Métodos HTTP

Método CRUD Idempotente
GET Read ✔
POST Create ✖

Ejemplo de Response

{
  "data": {
    "id": 123,
    "name": "Ejemplo REST",
    "links": {
      "self": "/api/v1/resources/123",
      "related": "/api/v1/resources/123/subresources"
    }
  }
}

SOAP: Protocolo Empresarial

Características Clave

  • Basado en XML: Estructura estricta
  • Protocolo independiente: HTTP, SMTP, etc.
  • WS-* Standards: Seguridad, transacciones
  • Contract-First: WSDL define el contrato

Ventajas vs REST

  • Seguridad integrada (WS-Security)
  • Transacciones ACID
  • Mejor para sistemas legacy

Ejemplo de Mensaje

<soap:Envelope>
  <soap:Header>
    <wsse:Security>...</wsse:Security>
  </soap:Header>
  <soap:Body>
    <GetUserRequest>
      <UserId>123</UserId>
    </GetUserRequest>
  </soap:Body>
</soap:Envelope>

JWT: Autenticación Moderna

Estructura

header.payload.signature

  • Header
  • Payload 
{
  "alg": "HS256",
  "typ": "JWT"
}
{
  "sub": "1234567890",
  "name": "John Doe",
  "iat": 1516239022
}
Implementación Node.js
const jwt = require('jsonwebtoken');
const token = jwt.sign(
  { userId: 123 }, 
  'clave-secreta',
  { expiresIn: '1h' }
);
Flujo de Autenticación
  1. Cliente envía credenciales
  2. Servidor genera JWT
  3. Cliente usa JWT en headers
  4. Servidor valida firma
